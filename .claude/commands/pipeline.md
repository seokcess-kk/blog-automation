# /pipeline

> 전체 파이프라인 실행 명령어

---

## 설명

키워드 입력부터 발행까지 전체 파이프라인을 실행합니다.

---

## 사용법

```
/pipeline <키워드> [옵션]
```

### 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--mode` | conservative | 발행 모드 |
| `--dry-run` | false | 발행 제외 테스트 |
| `--images` | 5 | 이미지 생성 수 |
| `--skip-publish` | false | 발행 단계 스킵 |

---

## 파이프라인 단계

```
┌──────────────────────────────────────────────────────────┐
│ 1. Analyze                                               │
│    키워드 → SERP 수집 → 콘텐츠 파싱 → 패턴 추출          │
│    [Scrapling]                                           │
└────────────────────────┬─────────────────────────────────┘
                         │ pattern_id
                         ▼
┌──────────────────────────────────────────────────────────┐
│ 2. Generate                                              │
│    패턴 → 프롬프트 → 원고 생성 → 의료광고법 검증          │
│    [Claude API]                                          │
│                                                          │
│    프롬프트 → 이미지 생성 → EXIF 삽입                     │
│    [Nano Banana Pro]                                     │
└────────────────────────┬─────────────────────────────────┘
                         │ draft_id
                         ▼
┌──────────────────────────────────────────────────────────┐
│ 3. Publish                                               │
│    세션 확인 → 에디터 조작 → 발행 → 결과 저장             │
│    [Playwright]                                          │
└──────────────────────────────────────────────────────────┘
```

---

## 실행 예시

### 기본 실행

```
/pipeline "강남 피부과"
```

### Dry-run (발행 제외)

```
/pipeline "강남 피부과" --dry-run
```

### 분석 + 생성만

```
/pipeline "강남 피부과" --skip-publish
```

---

## 출력 예시

```markdown
# 파이프라인 실행 결과

## 키워드: 강남 피부과
## 모드: conservative
## 시작: 2025-03-10 14:00:00

---

### 1. Analyze ✅
- 수집 URL: 5개
- 평균 글자수: 2,500자
- 평균 이미지: 7장
- 소요 시간: 45초

### 2. Generate ✅
- 제목: "강남 피부과 추천 TOP 5 - 2025 최신 정보"

- 발행 시간: 2025-03-10 14:05:30
- 소요 시간: 180초

---

## 총 소요 시간: 5분 45초

## 결과
- keyword_id: uuid-xxx
- pattern_id: uuid-yyy
- draft_id: uuid-zzz
- post_url: https://blog.naver.com/blogid/123456789
```

## 에러 발생 시

```markdown
# 파이프라인 실행 결과

## 키워드: 강남 피부과
## 상태: ❌ 실패

---

### 1. Analyze ✅
### 2. Generate ❌

**에러:**
- 유형: MedicalAdViolationError
- 메시지: Critical 위반 발견 (2건)
- 위치: content_generator.py:145

**위반 내용:**
1. "100% 효과" - 법56조②8호
2. "완치" - 법56조②2호

**해결 방법:**
1. /check-medical로 상세 확인
2. 프롬프트 수정
3. /pipeline 재실행
```

---

## 관련 명령어

- `/analyze`: 분석만 실행
- `/generate`: 생성만 실행
- `/publish`: 발행만 실행
- `/check-medical`: 의료광고법 검증

---

## 관련 에이전트

- `pipeline-runner`: 배치 실행
- `auto-error-resolver`: 에러 자동 해결
- `debug-publisher`: 발행 디버깅

---

## 주의사항

- 의료 키워드는 conservative 모드 필수
- 하루 5개 초과 발행 금지
- Critical 위반 시 발행 자동 중단
- 실패 시 `auto-error-resolver` 에이전트 호출
